Web-PHP（1）弱类型

PHP本身的安全问题一直都是一个问题，它存在一个特性，也就是弱类型，弱类型的语言对变量的数据类型没有限制，你可以在任何地时候将变量赋值给任意的其他类型的变量，同时变量也可以转换成任意地其他类型的数据。PHP正是如此，导致在进行类型转换的时候会存在很多意想不到的问题。

当然，php虽然属于弱类型语言，但也存在强制定义数据类型的办法。

  • == 为松散比较 只比较值，不比较数据类型
  • === 为严格比较，不仅比较值也比较类型，可以看作是强语言的强制数据类型要相同

类型转换问题

Hash比较

PHP在处理字符串的时候把e认为是科学计数法，所以0e100被认为和0相等。（0乘任何数都为0）

十六进制转换

字符串是0x开头的时候，PHP会将此字符串解析成为十进制然后再进行比较，比如说0x1240被认为123456相等。

类型转换

当一个字符串欸当作一个数值来取值，其结果和类型如下：如果该字符串没有包含’.’,’e’,’E’ 并且其数值值在整形的范围之内该字符串被当作 int 来取值，其他所有情况下都被作为 float 来取值，该字符串的开始部分决定了它的值，如果该字符串以合法的数值开始，则使用该数值，否则其值为 0。

Bool类型

在PHP中任何类型的值, 与bool比较都会被转化成bool比较当转换为 bool 时，以下值被认为是 false：

  • 布尔值 false 本身
  • 整型值 0（零）
  • 浮点型值 0.0（零）-0.0（零）
  • 空字符串 ""，以及字符串 "0"
  • 不包括任何元素的数组
  • 单位类型 NULL（包括尚未赋值的变量）
  • 内部对象的强制转换行为重载为 bool。

所有其它值都被认为是 true
