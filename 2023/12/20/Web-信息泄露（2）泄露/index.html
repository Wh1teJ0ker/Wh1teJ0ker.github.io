Web-信息泄露（2）泄露

前言

对一些常见泄露做一个简单的总结

前端问题

首先要有一个认知，所有在前端的认证都是不安全的，所以重要信息一定不要放在前端。

包括前端js敏感信息泄露，或者是一些身份登录认证，或者是验证码之类的等等都不要在前端进行验证。

一旦只在前端，势必有绕过的手段。

泄露问题

Git泄露

当前大量开发人员使用git进行版本控制，对站点自动部署。如果配置不当,可能会将.git文件夹直接部署到线上环境。这就引起了git泄露漏洞。

实际上就是直接访问/.git存在，说明存在该问题，当然，现在已经有了现成的利用工具。

利用工具BugScanTeam/GitHack: .git 泄漏利用工具，可还原历史版本 (github.com)

常用命令

SVN泄露&HG泄露

kost/dvcs-ripper: Rip web accessible (distributed) version control systems: SVN/GIT/HG… (github.com)

总结

这些只是做了一个简单的罗列，实际中根据开发人员的习惯可能还会有太多的可能性了，一个例如dirsearch之类的扫描工具+一个大字典对Get请求进行fuzz，查看反馈的敏感信息更为有效。了解所谓的信息泄露基础漏洞只是为了对此有一个更清晰有效的认知。鄙人拙见，不喜勿喷。
